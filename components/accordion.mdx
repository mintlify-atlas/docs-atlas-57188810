---
title: "Accordion: Collapsible content sections for Svelte 5"
description: "A headless accordion component for Svelte 5 that organizes content into collapsible sections with single-open or multi-open selection modes."
---

The Accordion component organizes content into collapsible sections, helping users focus on specific information without being overwhelmed by visual clutter. It supports both single-open and multi-open modes, full keyboard navigation, and smooth transitions via CSS variables or Svelte's built-in transition system.

## Installation

```svelte
<script lang="ts">
  import { Accordion } from "bits-ui";
</script>
```

## Usage

```svelte
<script lang="ts">
  import { Accordion } from "bits-ui";
</script>

<Accordion.Root type="single">
  <Accordion.Item value="item-1">
    <Accordion.Header>
      <Accordion.Trigger>Item 1 Title</Accordion.Trigger>
    </Accordion.Header>
    <Accordion.Content>
      This is the collapsible content for item 1.
    </Accordion.Content>
  </Accordion.Item>
  <Accordion.Item value="item-2">
    <Accordion.Header>
      <Accordion.Trigger>Item 2 Title</Accordion.Trigger>
    </Accordion.Header>
    <Accordion.Content>
      This is the collapsible content for item 2.
    </Accordion.Content>
  </Accordion.Item>
</Accordion.Root>
```

### Multiple open items

Set `type="multiple"` to allow more than one item to be open at the same time.

```svelte
<Accordion.Root type="multiple">
  <!-- items -->
</Accordion.Root>
```

### Two-way binding

Bind the `value` prop to react to open/close changes externally.

```svelte
<script lang="ts">
  import { Accordion } from "bits-ui";
  let myValue = $state<string[]>([]);
</script>

<button onclick={() => (myValue = ["item-1", "item-2"])}>
  Open items 1 and 2
</button>

<Accordion.Root type="multiple" bind:value={myValue}>
  <Accordion.Item value="item-1"><!-- ... --></Accordion.Item>
  <Accordion.Item value="item-2"><!-- ... --></Accordion.Item>
</Accordion.Root>
```

### Svelte transitions

Use `forceMount` with the `child` snippet to apply Svelte transitions to `Accordion.Content`.

```svelte
<script lang="ts">
  import { Accordion } from "bits-ui";
  import { slide } from "svelte/transition";
</script>

<Accordion.Content forceMount>
  {#snippet child({ props, open })}
    {#if open}
      <div {...props} transition:slide={{ duration: 200 }}>
        Animated accordion content.
      </div>
    {/if}
  {/snippet}
</Accordion.Content>
```

## API Reference

### Accordion.Root

The root container that manages overall accordion state.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `type` | `"single" \| "multiple"` | — | **Required.** Controls whether one or multiple items can be open at a time. |
| `value` | `string` (single) or `string[]` (multiple) | `""` / `[]` | The currently open item(s). Bindable. |
| `onValueChange` | `(value: string \| string[]) => void` | — | Callback fired when the open value changes. |
| `disabled` | `boolean` | `false` | Disables all accordion items when `true`. |
| `loop` | `boolean` | `true` | Whether keyboard navigation loops from last to first item. |
| `orientation` | `"horizontal" \| "vertical"` | `"vertical"` | The orientation of the accordion for keyboard navigation. |

### Accordion.Item

Represents a single collapsible section within the accordion.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | auto-generated | Unique identifier for this item, used to track open state. |
| `disabled` | `boolean` | `false` | Prevents user interaction with this item when `true`. |

**Data attributes**

| Attribute | Values |
|-----------|--------|
| `data-state` | `"open"` \| `"closed"` |
| `data-disabled` | Present when the item is disabled. |

### Accordion.Header

Renders the visible heading element for an accordion item.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `level` | `1 \| 2 \| 3 \| 4 \| 5 \| 6` | `3` | The ARIA heading level applied via `aria-level`. |

### Accordion.Trigger

The interactive button that toggles the visibility of `Accordion.Content`. Renders a `<button>` element.

No component-specific props beyond inherited HTML button attributes.

**Data attributes**

| Attribute | Values |
|-----------|--------|
| `data-state` | `"open"` \| `"closed"` |
| `data-disabled` | Present when the parent item is disabled. |

### Accordion.Content

The collapsible body of an accordion item.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `forceMount` | `boolean` | `true` | When `true`, keeps the content in the DOM even when closed. Required for Svelte transitions. |
| `hiddenUntilFound` | `boolean` | `false` | Marks content with `hidden="until-found"` so browsers can reveal it during a find-in-page search. Overrides `forceMount`. |

**CSS variable**

| Variable | Description |
|----------|-------------|
| `--bits-accordion-content-height` | The current height of the content element, useful for CSS transitions. |

**Child snippet props**

| Prop | Type | Description |
|------|------|-------------|
| `open` | `boolean` | Whether this accordion item is currently open. |

## Accessibility

The Accordion follows the [WAI-ARIA Accordion pattern](https://www.w3.org/WAI/ARIA/apg/patterns/accordion/).

- `Accordion.Header` renders a heading element with the appropriate `aria-level`.
- `Accordion.Trigger` renders a `<button>` with `aria-expanded` and `aria-controls` managed automatically.
- `Accordion.Content` receives `role="region"` and `aria-labelledby` pointing to the associated trigger.

## Keyboard interactions

| Key | Action |
|-----|--------|
| `Enter` / `Space` | Toggles the focused accordion item open or closed. |
| `ArrowDown` | Moves focus to the next trigger (vertical orientation). |
| `ArrowUp` | Moves focus to the previous trigger (vertical orientation). |
| `ArrowRight` | Moves focus to the next trigger (horizontal orientation). |
| `ArrowLeft` | Moves focus to the previous trigger (horizontal orientation). |
| `Home` | Moves focus to the first trigger. |
| `End` | Moves focus to the last trigger. |
| `Tab` | Moves focus to the next focusable element. |
