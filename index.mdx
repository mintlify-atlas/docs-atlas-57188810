Bits UI provides 41 fully headless, WAI-ARIA compliant component primitives for Svelte 5. Each component ships completely unstyled so you can bring your own design system — whether that's Tailwind CSS, plain CSS, or any other approach. Every component exposes a `class` prop, state-based `data-*` attributes, and CSS variables so you have complete control.

## Why Bits UI?

Bits UI is built for developers who want complete control over their component design without sacrificing accessibility or keyboard navigation. Components follow WAI-ARIA patterns out of the box — focus management, screen reader support, and keyboard interactions are all handled for you.

## Quick install

<CodeGroup>
```bash npm
npm install bits-ui
```

```bash pnpm
pnpm add bits-ui
```

```bash yarn
yarn add bits-ui
```

```bash bun
bun add bits-ui
```
</CodeGroup>

## A taste of the API

```svelte
<script lang="ts">
  import { Accordion } from "bits-ui";
</script>

<Accordion.Root type="single">
  <Accordion.Item value="item-1">
    <Accordion.Header>
      <Accordion.Trigger class="flex w-full items-center justify-between p-4">
        What is Bits UI?
      </Accordion.Trigger>
    </Accordion.Header>
    <Accordion.Content class="px-4 pb-4">
      A headless component library for Svelte 5 that gives you full control
      over styling while handling accessibility for you.
    </Accordion.Content>
  </Accordion.Item>
</Accordion.Root>
```

<Tip>
  Bits UI requires Svelte 5. If you're on Svelte 4, see the [v0 documentation](https://v0.bits-ui.com) or [migrate to v1+](/migration-guide).
</Tip>
